格式说明：

返回值类型：

  • 有返回值：八种基本数据类型.引用数据类型
  • 无返回值：void

**方法名：**见名知意，小驼峰命名法

**参数：**由数据类型和变量名组成，多个参数之间用逗号隔开

**方法体：**完成功能的代码块

**return：**把数据返回给调用者

  • 有返回值，return后面的数据需要和返回值类型保持一致
  • void(无返回值)，省略return不写
10.2 定义方法时的两个明确

  • 明确参数：主要是明确参数的类型和数量

  • 明确返回值类型：主要是明确方法操作完毕之后是否有数据返回，如果没有，写void；如果有，写对应的数据类型

10.3 调用方法
  • void类型的方法，直接调用即可
  • 非void类型的方法，推荐用变量接收调用

11.方法的注意事项

11.1 注意事项
  • 方法需要先定义，后调用
  • 方法调用时传递的实际参数的个数和数据类型要与方法定义位置的形式参数个数及数据类型一致
  • 方法是平级关系，不能嵌套定义
  • 方法的返回值类型为void，表示该方法没有返回值，没有返回值的方法可以省略return语句不写，如果要编写return，后面不能跟具体的数据。
  • return语句下面，不能编写代码，因为永远执行不到，属于无效的代码
11.2 return
  • return ：是一个关键字，表示方法的返回，这个方法执行到return就表示要返回了
  • 使用范围：在方法体中
  • 作用：结束方法
  • 针对于有返回值类型的方法：结束方法并返回具体的数据或者变量

注意：

  • return 关键字后边不应该再有执行语句
  • 一个方法只能有一个返回值
  • 如果说是 void，return可写.可不写，写return；后面不能跟具体的数据

​ 1.默认方法
​ 问题:
​ 1.定义格式
​ 2.作用
​ 3.使用方式
​ 答案:
​ 1.定义格式:
​ public default 返回值类型 方法名(形式参数列表){
​ 方法体;
​ }
​ 2.作用: 解决接口升级问题:
​ 如果直接在接口中添加抽象方法,导致所有的接口实现类都要强制重写该方法,会影响程序的功能

3.使用方式
实现类对象名.默认方法名(实际参数);

2.静态方法
问题:
1.定义格式
2.作用
3.使用方式
答案:
1.定义格式:
public static 返回值类型 方法名(形式参数列表){
方法体;
}
2.作用:解决接口升级问题:
如果接口中书写的默认方法(default),所有的实现类都需要进行使用,需要创建每一个实现类对象进行调用

​ 定义为静态方法,不需要创建实现类对象,直接在使用位置通过
​ [接口名.]调用方式,简化调用方式,同时节约内存,减少实现类对象的频繁创建
​ 3.使用方式:
​ 接口名.静态方法名(实际参数);
​ 问题:为什么只能使用该方式进行调用?
​ 1.静态方法不能被重写
​ 2.在继承关系下,如果使用子类对象调用静态方法
​ 如果子类中存在静态方法,优先使用子类
​ 如果子类中不存在静态方法,调用super空间中父类静态方法
​ 3.在实现关系下,[子类是不能从implements空间查找静态方法],implements空间中只能使用接口名进行访问

​ 建议:静态都是用类名.调用

二.多态

1.多态的概述

​ 问题:
​ 1.什么是多态
​ 2.多态前提条件
​ 答案:
​ 1.多态指的是同一个对象在同一时刻的不同表现形式
​ 例如:
​ 对于猫个体,同一时刻有两种不同表现形式
​ 1.猫是猫
​ 2.猫是一种动物

​ 此时只创建一个猫的对象

​ 2.多态的前提条件
​ 1.有继承(extends)或者实现(implements)关系
​ 2.要有方法重写
​ 3.要有父类引用指向子类对象或者接口引用指向实现类对象
​ 理解:使用父类/接口类型变量接收子类/实现类对象的地址
​ 例如:
​ Fu f = new Zi();
​ Inter i = new InterImpl();

2.多态成员访问特点

​ 1.构造方法
​ new关键字调用哪个构造方法,就是创建哪个类的对象
​ 2.成员变量
​ 编译看左边,运行看左边
​ 因为成员变量不存在重写
​ 3.成员方法
​ 编译看左边,运行看右边
​ 成员方法存在重写,此时运行看子类是否重写父类方法
​ 重写:执行子类重写后的方法
​ 没有重写:执行父类方法

​ Fu f = new Zi();

​ f.变量 – 使用什么类型去访问,就在当前类型锁表示的空间进行查找

​ 编译时期看 Fu f;只知道变量是Fu类型

​ 要想编译通过,必须保证使用f访问的内容是存在的,才能通过编译

​ 运行时期,编译通过,执行new Zi(),会产生方法重写

​

3.多态的好处和弊端

​ 问题:
​ 1.多态的好处
​ 2.多态的弊端
​ 答案:
​ 1.好处:
​ 将父类类型/接口类型定义为方法的形式参数/返回值类型
​ 1.提高方法的兼容性(可以接收所有子类/实现类对象)
​ 2.减少重载方法的书写(System.out.println();)

​ println();//基本类型 + char[] + String + Object

​ 2.弊端:
​ 不能使用子类/实现类的特有功能,只能使用父类/接口中存在的成员

​ (只能使用编译时期可以找得到的,成员变量–看父类/接口,成员方法–看父类/接口)

4.多态中的转型

​ 问题:
​ 1.转型的分类
​ 2.转型的格式
​ 答案:
​ 1.分类:
​ 1.自动类型提升(向上转型)
​ 2.强制类型转换(向下转型)
​ 2.格式:
​ 1.自动类型提升:
​ 1.使用父类类型变量接收子类对象
​ Fu f = new Zi();
​ 2.使用接口类型变量接收实现类对象
​ Inter i = new InterImpl();
​ 2.强制类型转换:
​ Fu f = new Zi();
​ Zi z = (Zi)f;
​ 或者
​ Inter i = new InterImpl();
​ InterImpl ii = (InterImpl)i;

应用场景:

当方法形式参数定义为接口/父类类型,此时接收子类对象已经自动类型提升为接口/父类类型

在方法体中,想要使用子类/实现类[特有]功能,必须进行强制类型转换

非场景下使用:

如果是正常创建对象使用,推荐直接创建当前类对象即可

子类/实现类对象,不使用多态

​

5.多态中的转型风险及解决方案

​ 问题:
​ 1.强制类型转换的目的
​ 2.多态强制类型转换会遇到什么风险
​ 3.如何解决风险
​ 答案:
​ 1.目的:
​ 为了使用子类/实现类特有的内容
​ 2.遇到的风险:
​ 强制类型转换时,会产生目标类型和转换类型不匹配情况
​ 造成ClassCastException
​ 例如:
​ 父类-Perosn:
​ 子类1:Student
​ 子类2:Teacher
​ Person p = new Student();
​ Teacher t = (Teacher)p;
​ 此时:p实际对象时Student,强转为Teacher
​ 类型不匹配
​ 3.解决:
​ 使用instanceOf关键字进行转型前判断
​ 变量名 instanceOf 目标类型
​ 结果为true:类型匹配,可以进行强转为目标类型的操作
​ 结果为false:类型不配,不能进行强转
​ 例如:
​ if(p instanceOf Student){
​ Student s = (Student)p;
​ }

如何去快速识别当前使用方法是不是多态形式:

看返回值类型和形式参数列表即可

将父类/接口类型定义为方法的形式参数/返回值类型 – 多态使用
